# 🌤️ Mood & Weather App

> A full-stack web application that combines real-time weather data with personal mood tracking — built from scratch over 4 weeks as a hands-on introduction to modern web development.

---

## 📌 Project Overview

This project is a beginner-to-intermediate learning journey that results in a fully functional mood and weather tracking app. Users can log their current mood, view live weather data for their location, and reflect on how the weather affects how they feel.

The project is structured into four weekly milestones, each introducing a new layer of the development stack.

---

## 🗓️ Development Roadmap

### Week 01 — Version Control & Project Setup
**Goal:** Establish a solid foundation using Git and GitHub.

- Initialize a local Git repository
- Create and configure a GitHub remote repository
- Learn the core Git workflow: `init`, `add`, `commit`, `push`, `pull`, `clone`...
- Write the initial `README.md` (you're reading it!)
- Understand branching basics and why version control matters for every developer

**Outcome:** A live GitHub repository with a clean commit history and project documentation in place.

---

### Week 02 — Static Front End (HTML & CSS)
**Goal:** Build the visual interface of the app before adding any logic.

- Structure the app layout using semantic HTML5
- Style the UI with CSS (custom properties, flexbox/grid, responsive design)
- Design key UI components:
  - Mood selector (e.g., emoji-based or slider input)
  - Weather display card
  - Daily log / history section
- Focus on clean, readable markup and a consistent visual language

**Outcome:** A fully styled, static front end that looks like a real app — even if nothing works yet.

---

### Week 03 — JavaScript Logic & Interactivity
**Goal:** Bring the interface to life with dynamic functionality.

- Add, edit, and delete mood entries using vanilla JavaScript
- Manipulate the DOM to update the UI in real time
- Store mood data locally using `localStorage`
- Handle user events (clicks, form submissions, input changes)
- Introduce basic state management concepts

**Outcome:** A fully interactive mood tracker that works entirely in the browser, without a backend.

---

### Week 04 — Backend Server & API Integration
**Goal:** Connect the app to the real world with a server and live data.

- Set up a lightweight backend server (Node.js + Express)
- Integrate a weather API (e.g., [OpenWeatherMap](https://openweathermap.org/api)) to fetch live weather data based on user location
- Handle API requests securely (keeping API keys server-side)
- Connect the front end to the backend via `fetch` requests
- Deploy the app or prepare it for deployment

**Outcome:** A fully functional full-stack web app that displays real-time weather alongside mood entries.

---
